Repositório de tarefa prática da formação EBAC, mantido como registro de evolução técnica, organização de entrega e uso do GitHub como ferramenta de rastreabilidade.

O conteúdo deste repositório é simples, mas seu valor está no processo: registrar uma entrega, preservar o material de apoio e manter contexto suficiente para que a tarefa possa ser entendida posteriormente.

## Objetivo do repositório

Este repositório foi criado para armazenar uma entrega prática associada ao módulo de Qualidade de Software da EBAC. Ele funciona como evidência de estudo e como registro de uma etapa da formação.

Objetivos principais:

- manter a tarefa versionada;
- centralizar o material de apoio;
- registrar a entrega em ambiente público;
- documentar o propósito do repositório;
- tratar uma atividade pequena com organização profissional.

## Como analisar o material

1. Abra o PDF de apoio.
2. Identifique o objetivo da atividade.
3. Leia os critérios ou orientações do exercício.
4. Relacione o material com práticas de QA, como análise, documentação e organização de evidências.

## Leitura de QA

Em qualidade de software, nem toda entrega é automação. Muitas vezes, o trabalho começa com análise, interpretação de requisitos, leitura crítica de material, documentação e organização de evidências.

Este repositório registra esse lado do trabalho: a capacidade de estruturar uma entrega, preservar contexto e manter rastreabilidade.

## Resultado técnico

A proposta deste repositório é simples, mas importante: registrar prática, manter histórico limpo e comunicar o propósito da entrega. Em QA, essa disciplina aparece em bug reports, evidências, planos de teste e documentação de automação.

## Competências evidenciadas

- Organização de artefatos.
- Uso de GitHub como repositório de entrega.
- Documentação contextual.
- Rastreabilidade de estudo.
- Visão de QA além da automação.

## Possíveis evoluções

- Adicionar um resumo do conteúdo do material de apoio.

## Conclusão

Este repositório mostra que qualidade também é processo. Antes de testar ou automatizar, é preciso organizar, entender, registrar e comunicar. Esse cuidado aparece aqui em uma entrega simples e se conecta diretamente a práticas profissionais de QA.
